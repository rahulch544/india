1. Create /prepos/vol directory
   a. Login to your host where forms is running as arudev
   b. mkdir -p /scratch/arudev/prepos
        If arudev does not have permissions
          i. login as root
         ii. mkdir -p /scratch/arudev/prepos
        iii. chown -R "arudev:<get arudev gid>" /scratch/arudev
   c. Login as root
     i. ln -s /scratch/arudev/prepos /prepos
2. Register your host in DB with arudev account
   a. source your env files
   b. sqlplus aru/aru @$ISD_HOME/log/register_host.sql ==> use plsql api to register host
        Note down the host_id returned by the above script

====== For downloading patch following setup is requried =============        
3. Updating the Patch repos location
   a. source your env files
   b. sqlplus aru/aru @$ISD_HOME/log/get_location.sql
         at the prompt enter ARU# . this will return location_id, location_path and file_name
   c. login as arudev/arudev ( unix login ) to your host and mkdir -p <location_path from previous step>
4. download the patch from production and copy it to <location_path from step 1.a>
5. Update $ISD_HOME/log/upd_location_host.sql, add your host_id from step 2 ( Register your host )
5. SQLplus to aru/aru  and run the script
   $ISD_HOME/log/upd_location.sql , enter <location_id from step1.a>. . This is updating the patch location to use your host
6. retry failed request or submit  a new request
